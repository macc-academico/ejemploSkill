Entendimiento Profundo

¿Cómo encajan las Skills de IA aquí?

Las habilidades modulares son la clave para que cualquier agente (Claude, GPT, Gemini) ejecute el ecosistema .agent de forma inteligente.

Revelación Progresiva

El agente no lee todas las carpetas a la vez. Lee las descripciones y solo activa el código cuando la tarea lo requiere, ahorrando tokens.

Interoperabilidad

Utilizando la estructura estándar `.agent/skills/`, tu configuración es compatible y portátil tanto para Claude Code, GPTs, Gemini y editores como Cursor.

Autonomía Controlada

Tus archivos de reglas (`rules/`) y herramientas (`tools/`) delimitan el comportamiento del agente, asegurando calidad en cada compilación.

El Cerebro

¿Qué es la carpeta .agent?

Es la caja negra y memoria de desarrollo del repositorio. Centraliza todo lo que el asistente de IA necesita saber para ser productivo en tu proyecto.

  • Prompts: Guías explícitas de arquitectura y estilo de código.
  • Rules: Reglas inviolables de seguridad, SEO y calidad de código.
  • Memory: Historial y base de conocimiento persistente del proyecto.
Consistencia

Plantillas de Código (.tpl)

El estándar para evitar la pereza de la IA y asegurar que siga nuestras convenciones en lugar de inventar archivos aleatorios.

  • COMPONENT.html.tpl: Estructura HTML5 semántica y responsive enlazada.
  • COMPONENT.css.tpl: Estilos Glassmorphic limpios con variables CSS.
  • COMPONENT.js.tpl: Interactividad segura encapsulada bajo DOMContentLoaded.
Modularidad

¿Qué son las Habilidades (Skills)?

Capacidades procedimentales bajo demanda escritas en Markdown con YAML frontmatter. Le confieren "superpoderes" específicos al agente.

  • Revelación Progresiva: Solo consume tokens cuando la skill es activada.
  • Interoperabilidad Universal: Estándar para Cursor, Claude Code, GPT y Gemini.
  • Ejemplo en este Proyecto: La Skill `presentacion-agent` que maneja esta misma página.
El Futuro

El Futuro del Desarrollo

Los desarrolladores ya no solo escriben código; ahora diseñan **Ecosistemas de Agentes** organizando plantillas, reglas y habilidades específicas.

  • Integración instantánea de nuevos programadores y asistentes de IA.
  • Arquitectura limpia, escalable y agnóstica de modelo de lenguaje.
